# DragonEyes Upgrade Plan — 인프라·서비스 업그레이드 로드맵

> **이 문서는 무엇인가?**
> 
> DragonEyes 운영에 필요한 **외부 서비스·인프라 업그레이드**를 한 곳에서 관리하는 문서입니다.
> SaaS 운영하면서 "언제 무엇을 업그레이드해야 하는지" 잊지 않도록 추적합니다.
> 
> **사용법**:
> - 새 서비스 가입할 때마다 등록
> - 업그레이드 검토 시점 도달하면 알림
> - 비용 누적 추적 → 회계 분석
> - 의사결정 시 한눈에 비교
>
> **버전 정보**
> - 문서 버전: v1.0
> - 작성일: 2026-05-10
> - 작성자: 최승현 (4U Solution)
> - 위치: `~/dragoneyes/docs/UpgradePlan_v1.md`

## 🎯 1. Supabase — 가장 중요한 업그레이드 대상

### 현재 상태 (2026-05-10 결제)

- **플랜**: Pro
- **비용**: $25/월
- **혜택**: 7일 자동 백업, 100K MAU, 8GB DB, 250GB egress

### Pro → Team 업그레이드 검토 조건

다음 중 **하나라도 해당되면** Team 검토:

| 조건 | 해당 시점 (예상) |
|---|---|
| ☐ 공공기관 입찰 시작 (SOC2/ISO 인증 요구) | 출시 6개월~1년 후 |
| ☐ 사용자 5만명 이상 | 2~3년 후 |
| ☐ 외부 보안 감사 받아야 함 | 출시 전 |
| ☐ 의료 데이터 처리 (HIPAA) | 해당 사업 진출 시 |
| ☐ 14일 이상 백업 필요 (데이터 중요도 ↑) | 1년 후 |
| ☐ Point-in-Time Recovery 필요 | 사고 1번 경험 후 |
| ☐ Priority Support + SLA 필요 | 글로벌 고객 확보 시 |

### Team 플랜 비용 영향

- $599/월 = 약 80만원/월 (Pro 대비 24배)
- 연간 $7,188 = 약 950만원
- **DragonEyes 매출 1억원 이상부터 부담 가능**

### MAU 종량제 (Pro에서 100K 초과 시)

```
$0.00325 per MAU 초과분
사용자 200,000명 → $25 + $325 = $350/월 (Team이 더 저렴해지는 분기점 검토)
```

### 결제 정보

- 결제일: 2026-05-10
- 카드: (좋아요님이 입력하신 카드 — 회계용으로 기록)
- 갱신: 매월 자동
- 영수증: Supabase Dashboard → Settings → Billing

### 현재 상태 (확인 필요)

- 좋아요님 Streamlit 앱 호스팅 중
- 환경변수 관리 (.env 동기화)
- Railway Cron 작업 예정 (자동 이메일, 모니터링 통계 일배치)

### 확인할 사항

```bash
# 좋아요님 Railway 대시보드에서 확인
- 현재 플랜: Free / Hobby / Pro / Team?
- 월 비용: ?
- 사용 중인 리소스: CPU, RAM, 네트워크
- 사용량 한도: 어떤 수준?
```

### 업그레이드 검토 조건

| 조건 | 해당 시점 |
|---|---|
| ☐ Free 플랜 사용량 한도 초과 | 사용자 증가 시 |
| ☐ Cron 작업 여러 개 동시 실행 필요 | 모니터링 통계 페이지 도입 후 |
| ☐ 더 많은 RAM 필요 (Streamlit 무거워짐) | 동시 사용자 100명+ |
| ☐ 다중 환경 필요 (staging/prod 분리) | 출시 전 |

### Railway 대안 (참고)

| 서비스 | 가격 | 장단점 |
|---|---|---|
| Railway | $5~/월 (Hobby) | 한국어 문서 적음, 한국 결제 OK |
| Vercel | 무료 (제한) ~ $20 | Streamlit은 별로 |
| AWS | 종량제 | 복잡, 가장 저렴 가능 |
| Heroku | $25~ | 옛날부터 표준 |
| Streamlit Community Cloud | 무료 | 비공개 앱은 제한 |

→ **권장**: 일단 Railway 유지. 나중에 비용 폭증하면 AWS 검토.

## 🎯 3. Anthropic API — Claude API

### 현재 상태

- Pay-as-you-go (사용한 만큼 결제)
- 모델: Claude Sonnet 4.5 (자동추천 모니터링 등)
- API 키 5/6에 회전 완료

### 사용량 추적 필요

```
일일 사용량 추적:
- 자동추천 모니터링: 1일 X회 호출
- 챗봇 (드래곤파더): 1일 Y회 호출
- 모니터링 통계 AI 인사이트 (예정): 1일 1회 (시스템 + 큰 고객 5~10개)

월 예상 비용:
- 현재: $XX/월
- 모니터링 통계 도입 후: 추가 $10~30/월
- 사용자 10배 증가 시: $XX × 10
```

### 업그레이드 검토 조건

| 조건 | 행동 |
|---|---|
| ☐ Rate limit 자주 도달 | Tier 상향 신청 |
| ☐ 월 비용 $500+ | Volume Discount 협상 |
| ☐ Enterprise 가격 검토 | 월 $5,000+ |

### 비용 절감 전략 (장기)

1. **캐싱** (Prompt Caching) — 같은 시스템 프롬프트 반복 호출 시 비용 절감
2. **모델 다운그레이드** — Sonnet → Haiku (간단한 분류 작업)
3. **배치 처리** — 실시간 X, 자정 일배치로 묶어서 처리 (이미 v2.1에서 결정)
