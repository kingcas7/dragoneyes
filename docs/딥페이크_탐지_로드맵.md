# 🎭 딥페이크 탐지 로드맵 (v1 — 2026-07-06)

## 핵심 결론

**"검색어만으로는 딥페이크를 못 잡는다"는 판단이 맞습니다.** 딥페이크 대응은 성격이 다른 두 문제로 나뉘고, 각각 다른 기술이 필요합니다.

| 문제 | 무엇을 잡나 | 필요한 기술 | 현재 상태 |
|---|---|---|---|
| ① 유통·거래 탐지 | 제작 의뢰, 판매, 능욕방 초대 등 **텍스트 흔적** | 키워드 검색 + AI 텍스트 분석 (기존 파이프라인) | ✅ 2026-07-06 적용 완료 |
| ② 콘텐츠 판별 | 영상/이미지 **자체가 합성물인지** | 미디어 포렌식 (프레임 단위 딥러닝 분석) | ⬜ 미착수 — 본 로드맵 |

## Phase A — 키워드 탐지 (완료)

`generate_recommend_keywords()`에 `deepfake` 카테고리 신설. 가해·유통자가 쓰는 표현(제작 의뢰, 지인능욕, 옷 벗기는 AI 앱, 텔레그램 합성방 등)을 검색해 **사람(모니터링 요원)이 검토·신고**하는 기존 워크플로에 태움.

- 드래곤파더 추천 버튼에 "🎭 딥페이크 추천" 추가
- 인벤토리 보충·주간 배정 라운드로빈에 `deepfake` 플랫폼 포함
- AI 분석 프롬프트 체크리스트 ⑩번(딥페이크/합성) + 분류 카테고리 `딥페이크` 추가

**한계**: 검색어는 "딥페이크를 팔거나 시키는 글"을 잡는다. 제목·설명에 아무 표시 없이 올라온 합성 영상 자체는 못 잡는다.

## Phase B — 미디어 포렌식 (콘텐츠 판별)

영상·이미지가 AI 합성물인지 판별하는 별도 기술 계층. 접근 옵션 3가지:

### B-1. 상용 탐지 API (권장 시작점)
| 서비스 | 특징 | 비고 |
|---|---|---|
| Hive AI (Deepfake Detection) | 이미지/영상 API, 건당 과금 | 연동 난이도 낮음 |
| Reality Defender | 멀티모달(영상·음성·이미지), 기업용 | NCMEC 계열 협력 이력 |
| Sensity AI | 딥페이크 특화, 모니터링 플랫폼형 | EU 기반 |
| 딥브레인AI (국내) | 딥페이크 탐지 솔루션, 경찰청 납품 이력 | 국내 레퍼런스·한국어 지원 |

**연동 형태**: 모니터링 요원이 검토하는 영상의 썸네일/대표 프레임을 탐지 API에 보내 "합성 확률 %"를 보고서에 첨부 → 최종 판단은 사람. 기존 `search_and_analyze()` 파이프라인에 프레임 추출 단계만 추가하면 됨.

### B-2. 오픈소스 자체 탐지
- 프레임 추출(yt-dlp + ffmpeg) → 얼굴 검출 → 탐지 모델(FaceForensics++ 계열, EfficientNet 기반 등) 추론
- 장점: 건당 비용 0 / 단점: GPU 인프라 필요, 최신 생성모델 대응 지속 재학습 필요, Railway 환경엔 부적합
- **판단: 초기엔 비권장.** 탐지 모델은 생성 기술과 군비경쟁이라 자체 운영 부담이 큼

### B-3. 출처 검증 (C2PA / Content Credentials)
- 콘텐츠 서명 표준(C2PA) 메타데이터 확인 — Adobe·OpenAI·Google 참여
- 아직 커버리지가 낮아 보조 수단. 메타데이터 파싱만이라 비용은 거의 0

## Phase C — 신고·대응 연계
- 심각도 4~5 딥페이크 판정 건 → 방심위 디지털성범죄 신고 / 경찰청 사이버수사대 연계 절차 문서화
- 피해자 지원: 디지털성범죄피해자지원센터(d4u.stop.or.kr) 연계 안내를 캠페인 교육자료에 포함 (기존 교육 컨텐츠에 이미 언급 있음)
