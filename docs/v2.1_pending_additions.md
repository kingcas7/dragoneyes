## ♿ 음성 안내 시스템 (시각장애인 접근성, 5/10 결정)

### 핵심 원칙

1. **기본값 OFF** — 사용자가 직접 토글로 ON/OFF
2. **분리된 페이지 X** — 같은 페이지에서 모드만 활성화
3. **모든 사용자에게 동일한 시스템** — "분리된 평등"이 아닌 "선택의 자유"

### 적용 대상 페이지

| 페이지 | 우선순위 | 음성 안내 내용 |
|---|---|---|
| 모니터링 | P0 | 페이지 진입 안내, 위험 발견 알림, 아이콘 설명 |
| 보고서 | P0 | 보고서 종류 안내, 다운로드 안내, 기간 선택 안내 |
| 사용자 정보관리 | P0 | 본인 정보 음성 읽기, 수정 항목 안내 |

### 기술 스택

- **Web Speech API** (브라우저 내장) — 추가 비용 없음, 한국어(ko-KR) 지원
- **ARIA 라벨** (HTML 표준) — NVDA, JAWS, VoiceOver 등 스크린리더 호환
- **Streamlit components** — Streamlit에서도 적용 가능

### 핵심 기능

1. **사이드바 토글**: "🔊 음성 안내" 켜기/끄기
2. **음성 속도 슬라이더**: 0.5배 ~ 2.0배
3. **키보드 단축키**: Alt+A (토글), Alt+M (메뉴), Alt+H (도움말)
4. **페이지 진입 시 안내**: "모니터링 페이지가 열렸습니다. 오늘 X건 분석 중..."
5. **포커스 시 음성**: Tab으로 이동하면 해당 항목 설명
6. **긴급 알림 우선순위**: 위험 발견 시 즉시 음성

### 설정 저장 위치

- `users.preferences` JSONB 컬럼에 저장
- 예: `{"voice_guide_enabled": true, "voice_speed": 1.0}`

### 음성 스크립트 작성 원칙

각 페이지마다 음성 안내 스크립트를 별도 파일로 관리:

- `accessibility/scripts/monitoring.py` — 모니터링 페이지 스크립트
- `accessibility/scripts/reports.py` — 보고서 페이지 스크립트
- `accessibility/scripts/user_profile.py` — 사용자 정보관리 스크립트

스크립트 작성 시 검토 필수:
- 시각장애인 사용자 1~2명 베타 테스트
- 한국시각장애인연합회 또는 관련 단체 자문 (선택)
- 음성 길이 적정선 (긴 안내는 사용자가 짜증)

| 단계 | 작업 | 시점 |

| 1 | accessibility 모듈 인프라 (announce 함수, 토글) | 다음주 |

### 주의사항

- **오용 시 역효과**: 어설픈 음성 안내는 시각장애인 단체의 비판 대상
- **테스트 필수**: 실제 시각장애인 사용자의 베타 테스트 없이 출시 금지
- **자동 감지 X**: 시각장애인이라고 자동 활성화하지 않음 (사용자 선택권 존중)
- **법정 시각장애 1~6급 모두 대응**: 저시력자도 사용 가능한 색상·폰트 크기 옵션
