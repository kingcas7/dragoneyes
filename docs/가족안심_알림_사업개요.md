# 🏡 드래곤아이즈 가족안심 알림 (가칭) — 사업 개요 v1

*2026-07-06 최승현 사장 구상 · 초안 정리*

## 한 줄 정의

시니어·장애인 모니터링 요원의 **하루 활동 여부**를 보호자에게 매일 알려주는 안부 구독 서비스.
어르신은 "아이들의 온라인 안전을 지키는 일"을 하고, 보호자는 그 활동 신호로 안부를 확인한다.

## 용어 원칙 — "가족"이 아니라 "보호자"

수신인을 처음부터 **보호자(guardian)**로 설계한다. 독거노인은 가족이 없거나 연락이 닿지 않는 경우가 있고, 그런 분들이야말로 이 서비스가 가장 필요한 대상이기 때문.

보호자 유형: **① 가족 ② 지자체 담당 공무원 ③ 복지기관·돌봄기관 담당자**
- **요원 등록 시 보호자 기재 필수** (최소 1명 — 가족 또는 담당 공무원)
- **가족은 원하는 만큼 복수 등록** (+ 추가 버튼, 형제·자매·손주 모두 가능)
- **요금 구분 (2026-07-06 확정)**: 가족 = **유료 opt-in**(월 1만원, 소개 알림에서 선택) / **담당 공무원·복지기관 = 무료 자동 수신** — 공공복지 지향. 독거 어르신의 안부 신호는 돈을 받고 파는 것이 아니라 공공에 제공한다

## 온보딩 플로우 — 소개 알림이 곧 영업 퍼널

요원이 가족을 등록하는 순간, 그 가족에게 **소개 알림**이 발송된다 (가족은 가입한 적 없어도 도달):

1. **통지**: "OOO님께서 드래곤아이즈 모니터링 활동을 시작하셨습니다"
2. **업무 브리핑**: 이 일이 무엇인지 — AI가 찾은 온라인 유해 의심 콘텐츠를 사람이 검증하는 활동
3. **지향점**: 어린이·청소년 온라인 보호 + 시니어 활동성 유지(안부 체크)
4. **유료 구독 제안**: "원하시면 평일 매일 카카오톡으로 OOO님의 활동 소식을 받아보실 수 있습니다 (월 1만원)" + 받게 될 메시지 예시("오늘은 OOO님께서…") 제시
5. **버튼**: [활동 소식 받기(구독)] / [수신 거부]

- 가족이 여럿이면 각자 개별 opt-in (딸은 구독, 아들은 거부 가능)
- **공무원·기관 보호자에겐 이 퍼널 없이** 등록+요원 동의 즉시 무료 수신 시작 (업무보고 톤)
- 소개 알림 = 고객획득비용(CAC) 0원의 내장 영업 채널
- **수신 거부 ≠ 연락처 파기 (2026-07-06 확정)**: 거부는 '일일 알림'만 중단. 연락처는 **비상연락망으로 계속 보유** — 목적 분리(①일일 알림=선택·유료 ②비상 안부확인 연락=필수)로 처리

## 안전 에스컬레이션 — 자동 알림 뒤의 인간 개입

**"일상의 안심은 구독, 생명의 안전은 무료"** — 안전 개입은 구독 여부와 무관하게 전원 무료.

1. 평일 무활동 1일 → 자동 알림 (구독 보호자에게)
2. 평일 3일 연속 무활동 → 상향 알림 (구독 보호자 + 공무원)
3. 지속 무응답 → **내부 '사용자 안전 모니터링 요원'(사람)이 요원 본인에게 직접 연락**
4. 본인 무응답 확인 시 → **등록된 모든 보호자에게 직접 전화** (수신 거부한 가족 포함 — 비상연락망 목적)

- 내부 안전요원은 별도 직무로 운영 (요원들의 무활동 대시보드 감시 + 전화 확인) — 이 역시 신규 일자리
- 이 안전망 덕에 유료 구독이 없는 독거 어르신도 동일한 보호를 받음 (공공복지 원칙의 실체)

### 소개 알림 시안 (가족용)

> [드래곤아이즈 가족안심]
> OOO님께서 드래곤아이즈 온라인 안전 모니터링 활동을 시작하셨습니다.
>
> 이 활동은 AI가 찾아낸 온라인 유해 의심 콘텐츠를 사람이 직접 확인해 어린이·청소년을 보호하는 일입니다. 어르신께는 규칙적인 활동과 사회적 역할이 되고, 가족분들께는 매일의 안부 신호가 됩니다.
>
> 원하시면 평일 저녁마다 OOO님의 활동 소식을 카카오톡으로 받아보실 수 있습니다 (월 10,000원).
> 예시: "오늘도 OOO님께서 온라인 안전을 지키는 수고를 해주셨어요."
>
> 또한 귀하는 OOO님의 **비상연락처(보호자)**로 등록되어, 장기간 활동이 확인되지 않는 경우 저희가 직접 연락드릴 수 있습니다. (알림 구독과 무관하게 적용되는 무료 안전 조치입니다)
>
> [활동 소식 받아보기] [알림 수신 거부]

## 핵심 반전 (왜 기존 안부 서비스와 다른가)

| | 기존 안부확인 (스마트플러그·케어콜·응급안전) | 가족안심 알림 |
|---|---|---|
| 어르신의 위치 | 감시당하는 대상 | **일하는 주체** (드래곤아이즈 요원) |
| 안부 신호의 근원 | 센서·AI 전화 | 의미 있는 활동의 부산물 |
| 어르신이 얻는 것 | 없음 | 역할 · 소득 · 인지활동 · 자부심 |
| 정서 | "확인당함" | "오늘도 수고하셨어요" |

같은 활동에서 수익이 두 번 발생: ① 모니터링 산출물(기존) ② 가족 구독(신규).

## 고객·가격

- **B2C**: 독거·시골 부모님(또는 장애인 가족)을 둔 4050 자녀 — **월 10,000원**
- **B2G(확장)**: 지자체 노인복지과 — "노인 디지털 일자리 + 안부확인"을 한 예산으로 (클로바 케어콜 구매 논리와 동일). 기존 관제센터·교육청 제안과 결합 가능

## 알림 설계 (확정 문구 — 사장님 원안)

- **활동한 날 (평일 저녁 발송)**
  > "OOO님께서 오늘도 우리를 위해서 온라인 안전을 지키는 수고를 해주셨어요. 가족분들도 함께 관심을 가져 주세요."
- **활동 없는 날**
  > "OOO님께서 오늘은 모니터링 업무를 수행하지 못하셨어요. 바쁜 일이 있는지 가족분들이 안부라도 부탁드려요."
- **평일 3일 연속 무활동 (단계 상향)**
  > "OOO님의 활동이 사흘째 확인되지 않고 있어요. 가족분들의 안부 확인을 부탁드립니다."
- **어르신 본인에게 주 1회 (금요일)**
  > "이번 주도 아이들의 온라인 안전을 지켜주셔서 감사합니다."
- **보호자가 공무원·기관 담당자인 경우 (문구 변형)**
  > 활동: "OOO님께서 오늘 온라인 안전 모니터링 활동을 정상 수행하셨습니다."
  > 무활동: "OOO님의 오늘 활동이 확인되지 않았습니다. 안부 확인을 권장드립니다."
  > (가족용 감성 톤 대신 업무 보고 톤 — 공무원 수신함에 맞게)

**원칙**: ① 수치(건수·실적) 절대 미포함 — 활동 여부만 ② 주말·공휴일 미발송(주말 로그인 차단 정책과 정합) ③ 실적 압박 톤 금지, 안부·감사 톤만 ④ 어르신 본인의 정보공유 동의 전제

## 데이터 설계 초안 (테이블 처음부터 보호자 유형 반영)

```sql
-- 보호자(수신인) — 요원 1명에 복수 등록
create table care_guardians (
  id             bigint generated always as identity primary key,
  worker_user_id uuid not null references users(id),   -- 시니어·장애인 요원
  guardian_type  text not null check (guardian_type in ('family','official','agency')),
                 -- family=가족, official=지자체 담당 공무원, agency=복지·돌봄기관
  name           text not null,
  relation       text,            -- 딸/아들/며느리 … 또는 직함(주무관 등)
  organization   text,            -- 공무원·기관인 경우: OO읍사무소, OO복지관
  email          text,
  phone          text,
  notify_channel text default 'alimtalk', -- alimtalk(기본) | email(폴백)
  notify_priority int default 1,         -- 다중 보호자 발송 순위
  -- 가족=유료 opt-in / 공무원·기관=무료 자동 (2026-07-06 확정)
  billing_type   text default 'paid' check (billing_type in ('paid','free')),
  status         text default 'registered' check (status in
                 ('registered',   -- 등록됨 (소개 알림 발송 전)
                  'intro_sent',   -- 소개 알림 발송됨 (가족: 응답 대기)
                  'subscribed',   -- 유료 구독 중 (가족) 또는 무료 수신 중 (공무원·기관)
                  'declined')),   -- 일일 알림 수신 거부 (비상연락망에는 유지 — 파기 금지)
  emergency_contact boolean not null default true,  -- 비상연락망 포함 (status와 무관, 항상 유지)
  subscribed_at  timestamptz,
  consent_at     timestamptz,            -- 요원 본인의 정보공유 동의 시각
  created_at     timestamptz not null default now()
);
-- 규칙: guardian_type='official'/'agency' → billing_type='free', 등록+요원동의 시 즉시 status='subscribed'
--       guardian_type='family' → 소개 알림(intro) 후 본인이 [구독]/[거부] 선택

-- 발송 이력 (무활동 연속일 계산·중복 방지)
create table care_notifications (
  id             bigint generated always as identity primary key,
  worker_user_id uuid not null,
  guardian_id    bigint not null references care_guardians(id),
  notify_date    date not null,
  kind           text not null default 'daily' check (kind in
                 ('intro',          -- 소개 알림 (가족 온보딩)
                  'daily',          -- 평일 활동/무활동 알림
                  'escalation',     -- 3일 연속 무활동 상향 알림
                  'weekly_thanks')),-- 요원 본인 금요일 감사 메시지
  activity       boolean,                -- 그날 활동 여부 (daily/escalation)
  streak_inactive int default 0,         -- 연속 무활동일 (3일 시 톤 상향)
  sent_at        timestamptz default now(),
  unique (guardian_id, notify_date, kind)
);
```

## 운영 원칙 (감수성)

- 시니어·장애인 요원에게는 **저위험 카테고리만 배정** (도박 스팸·가짜 건강정보·사칭 광고 등). 그루밍·성착취 의심 콘텐츠 배정 금지 — 배정 시스템이 카테고리 기반이라 설정으로 가능
- 시니어 대상 사기 콘텐츠 검증은 시니어 요원이 최적 (당사자가 검증자)

## 리스크

- 노동 압박으로 보이는 순간 붕괴 → 문구·주말 미발송·수치 미노출 원칙 고수
- 요원 계약 형태(근로/위촉/봉사+수당) 법률 검토 필요
- 무활동=이상신호의 오탐(여행·병원 등) → 어르신이 "휴무 예고" 남길 수 있는 간단한 장치 검토
- **제3자 연락처 등록 컴플라이언스**: 가족 연락처는 요원(본인)이 등록하는 제3자 개인정보 — 소개 알림에 유료 안내가 포함되면 광고성 정보로 판단될 수 있음. 대응: ① 소개 알림은 "활동 시작 통지 + 비상연락처 등록 고지" 중심 구성(구독 안내는 부가) ② [알림 수신 거부] 버튼 필수 — 단 **거부 시에도 연락처는 파기하지 않고 비상연락망 목적으로 보유** (수집 목적을 '일일 알림(선택)'과 '비상 안부확인 연락(필수)'으로 분리해 요원 등록 시 동의받고, 소개 알림에 비상연락처 등록 사실 고지) ③ 요원 등록 화면에 "보호자에게 안내 메시지가 발송되며 비상연락처로 등록됩니다" 고지 ④ 알림톡 템플릿 심사 시 정보성 인정 여부 사전 확인 ⑤ 요원 탈퇴·활동 종료 시에는 보호자 연락처 파기
